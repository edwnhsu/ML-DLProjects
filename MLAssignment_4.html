Explanation of Observed Differences in Accuracy¶

SVM Models: The differences in accuracy among the SVM models can be attributed to how each kernel handles the data's geometric distribution. The RBF kernel, which can manage complex, non-linear boundaries, performs best, suggesting non-linear relationships in the dataset. The polynomial kernel, offering a compromise between linear and complex non-linear mappings, outperforms the linear model but is less effective than RBF. The sigmoid kernel generally performs worse in many classification problems, as seen here, likely due to its output characteristics which are less suited for clear margin separation in SVM.

KNN: KNN performed comparably to the best SVM model, which suggests that proximity-based classification is very effective for this dataset. The high accuracy also indicates that local similarity is a strong indicator in this dataset, and that the features form well-clustered groups. The choice of 20 neighbors may have provided a good balance between bias and variance, enhancing its performance.

Conclusion¶

Strengths:

  • SVM with RBF Kernel: Best for capturing non-linear patterns, thus providing high accuracy where relationships between feature variables are complex.
  • Polynomial Kernel SVM: Useful when the data contains interactions of features at multiple scales (due to terms of different degrees).
  • Linear SVM: Efficient and straightforward, best when data is linearly separable or nearly so.
  • KNN: Strong performance without assumptions about data distribution; excels when data naturally clusters well.

Weaknesses: SVM with Sigmoid Kernel: Generally lower performance can reflect issues with how the kernel maps the decision boundary in higher-dimensional space. Linear and Polynomial Kernel SVM: May underperform if the true decision boundary is highly non-linear or complex beyond the polynomial degree set. KNN: Computationally intensive as the dataset grows; performance highly dependent on the choice of the number of neighbors and the distance metric.

def evaluate_algorithms(data, algorithms, cv=3, verbose=True):
    """Evaluates a list of collaborative-filtering algorithms on a dataset using cross-validation.

    Args:
        data (pandas.DataFrame): The dataset to use for evaluation.
        algorithms (list): A list of collaborative-filtering algorithm objects to evaluate.
        cv (int, optional): The number of cross-validation folds to use. Defaults to 3.
        verbose (bool, optional): Controls verbosity of the cross-validation process. Defaults to True.
    Returns:
        pandas.DataFrame: A DataFrame containing evaluation results and rankings
                          for each algorithm by Root Mean Squared Error (RMSE), Mean
                          Squared Error (MSE), Mean Absolute Error (MAE), Fraction
                          of Concordant Pairs (FCP).
    """

    benchmark = []
    # Iterate over algorithms
    for algorithm in algorithms:
        # Perform cross validation
        results = cross_validate(algorithm, data, measures=['RMSE', 'MAE', 'MSE', 'FCP'], cv=cv, verbose=verbose)

        # Get results & append algorithm name
        results_df = pd.DataFrame.from_dict(results).mean(axis=0)
        new_series = pd.Series([str(algorithm).split(' ')[0].split('.')[-1]], index=['Algorithm'])
        results_df = pd.concat([results_df, new_series], axis=0)
        benchmark.append(results_df)

    # Create benchmark DataFrame
    benchmark_df = pd.DataFrame(benchmark).set_index('Algorithm').sort_values('test_rmse')

    # Add ranking columns
    benchmark_df['RMSE Rank'] = benchmark_df['test_rmse'].rank(ascending=True).astype(int)
    benchmark_df['MAE Rank'] = benchmark_df['test_mae'].rank(ascending=True).astype(int)
    benchmark_df['MSE Rank'] = benchmark_df['test_mse'].rank(ascending=True).astype(int)
    benchmark_df['FCP Rank'] = benchmark_df['test_fcp'].rank(ascending=False).astype(int)

    return benchmark_df.round(4)

def analyze_predictions(predictions, trainset):
    """Analyzes predictions from the Surprise package, adding helpful columns.

    Args:
        predictions (list): A list of prediction objects as returned by
                            Surprise algorithms.
        trainset (Surprise.Dataset): The training dataset used to fit the model.

    Returns:
        pandas.DataFrame: A DataFrame with the following columns:

            * uid: User ID.
            * iid: Item ID.
            * actual: Real/true rating
            * predicted: Estimated rating.
            * details: JSON-string with prediction details:
                * actual_k: Number of neighbors used (if applicable).
                * was_impossible: Whether a prediction was possible.
            * Iu: Number of items the user has rated.
            * Ui: Number of users who rated the item.
            * error: Absolute error between the true and estimated rating.
    """

    # Create initial DataFrame
    df = pd.DataFrame(predictions, columns=['uid', 'iid', 'actual', 'predicted', 'details'])

    # Calculate Iu (number of items rated by the user)
    def get_Iu(uid):
        try:
            return len(trainset.ur[trainset.to_inner_uid(uid)])
        except ValueError:
            return 0

    df['Iu'] = df.uid.apply(get_Iu)

    # Calculate Ui (number of users who rated the item)
    def get_Ui(iid):
        try:
            return len(trainset.ir[trainset.to_inner_iid(iid)])
        except ValueError:
            return 0

    df['Ui'] = df.iid.apply(get_Ui)

    # Calculate error
    df['error'] = abs(df.predicted - df.actual)

    return df
